## MetricFlow 0.210.0 - April 24, 2026

### Breaking Changes

- Remove `MetricFlowEngine.get_measures_for_metrics` ([#1881](https://github.com/dbt-labs/metricflow/issues/1881))
- License Change - Version 0.209.0 and greater is covered by the Apache 2.0 license. ([#1905](https://github.com/dbt-labs/metricflow/issues/1905))

### Features

- Add new transformations to production from dsi that produce metrics from old-style measures. ([#387](https://github.com/dbt-labs/metricflow/issues/387))
- Generate nested CTEs to simplify SQL ([#1992](https://github.com/dbt-labs/metricflow/issues/1992))
- Add initial implementation of OSI converters ([#2017](https://github.com/dbt-labs/metricflow/issues/2017))

### Fixes

- Don't error for lack of time dimensions in manfest ([#1848](https://github.com/dbt-labs/metricflow/issues/1848))
- Pin to the latest non-dev release of dbt-semantic-interfaces. ([#1854](https://github.com/dbt-labs/metricflow/issues/1854))
- Use node_relation to render time spine table with proper quoting. ([#1875](https://github.com/dbt-labs/metricflow/issues/1875))
- Fixes a bug related to leaving out metric values in offset metrics, specifically if the queried grain is smaller than the offset grain. ([#1910](https://github.com/dbt-labs/metricflow/issues/1910))
- Properly apply time constraints on agg time dimensions for cumulative metrics. ([#1910](https://github.com/dbt-labs/metricflow/issues/1910))
- Fixes a bug causing errors for join_to_timespine metrics queried with date part. ([#1919](https://github.com/dbt-labs/metricflow/issues/1919))
- Add dunder prefix to column aliases for simple-metric inputs ([#1938](https://github.com/dbt-labs/metricflow/issues/1938))
- Render simple-metric inputs without a dunder prefix for the `WHERE` filter ([#1941](https://github.com/dbt-labs/metricflow/issues/1941))
- Improve handling of invalid metrics in metadata methods of `MetricFlowEngine` ([#1949](https://github.com/dbt-labs/metricflow/issues/1949))
- Fixes an issue where metric-level YAML filters are not applied for cumulative & conversion metrics. ([#1950](https://github.com/dbt-labs/metricflow/issues/1950))
- Fix logging setup in the CLI ([#1932](https://github.com/dbt-labs/metricflow/issues/1932))
- Fixes a bug where time dimensions utilizing the default time grain raise an error. ([#1955](https://github.com/dbt-labs/metricflow/issues/1955))
- Add memoization to `resolve_available_items` to prevent repeated expensive DAG traversals during query resolution with invalid group-by items ([#1964](https://github.com/dbt-labs/metricflow/issues/1964))
- Fix optimization of queries containing metrics with`fill_nulls_with` set ([#1963](https://github.com/dbt-labs/metricflow/issues/1963))
- Improve CTE generation via caching in `ComputeMetricsBranchCombiner` ([#1986](https://github.com/dbt-labs/metricflow/issues/1986))
- Prevent base-grain time dimensions from leaking into time-offset metric output ([#2004](https://github.com/dbt-labs/metricflow/issues/2004))
- Fix check for aggregation time dimensions in filters ([#2012](https://github.com/dbt-labs/metricflow/issues/2012))
- Fix bug with filters applied before aggregation in time-offset metrics ([#2016](https://github.com/dbt-labs/metricflow/issues/2016))
- Fix queries mixing aliased and non-aliased versions of the same dimension ([#2025](https://github.com/dbt-labs/metricflow/issues/2025))
